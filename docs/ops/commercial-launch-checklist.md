# Commercial Launch Checklist

## 当前状态

当前产品是可本地运行、可演示、可进入试点配置的商用 MVP。它已经具备：

- 商家配置后台。
- 商家专属咨询页。
- 浏览器语音输入和文字输入。
- 本地接待员 fallback。
- OpenClaw HTTP adapter。
- 线索摘要和后台查看。
- 飞书通知服务。
- PostgreSQL 迁移和 Docker Compose 本地数据库。

## 进入真实商家试点前必须补齐

### 1. OpenClaw

- `OPENCLAW_AGENT_URL`
- `OPENCLAW_API_KEY`
- Agent 输出 contract：`spoken` 和 `summary`
- 两个行业的 Agent 配置：摄影写真、家装设计
- 失败兜底规则：OpenClaw 超时或返回无效 JSON 时使用本地接待员 fallback

### 2. 通知通道

- 至少一个真实飞书机器人 webhook
- 通知接收人：老板、运营、技术值班
- 测试 `/api/notify/feishu/test`
- 确认无 webhook 时 lead 仍落库，通知错误可在后台定位

### 3. 试点商家资料

每个商家至少提供：

- 店名
- 行业
- 地址
- 服务范围
- 营业时间
- 联系电话
- 1-3 个核心套餐
- 10 条高频 FAQ
- 是否需要人工确认预约
- 小红书主页或笔记链接

### 4. 合规和用户告知

咨询页需要明确告知：

- 当前由 AI 接待助手提供初步咨询。
- 客户留下的信息会用于商家后续联系。
- 价格、档期、优惠以商家最终确认为准。
- 客户可要求人工跟进。

### 5. 试点验收

单个商家试点达标标准：

- 商家资料 30 分钟内完成配置。
- 至少 10 轮模拟咨询通过。
- 至少 20 次真实或半真实访问。
- 至少 5 条有效线索。
- 商家确认摘要可用于跟进。
- 没有编造价格、档期、优惠或结果承诺。

## Go / No-Go

满足以下条件才能对外收费：

- `npm test` 通过。
- `npm run build` 通过。
- `npm audit --audit-level=high` 通过。
- 至少 2 个真实商家完成试点。
- 至少 1 个商家愿意续用或付费。
- 通知失败、Agent 失败、无麦克风场景都有兜底。

未满足前，只能称为试点 MVP，不对外承诺稳定商用。
